## 二、数据概况

**文件**：`ref/consume.csv`，29,232 行 × 5 列，月频（每月1号），2013-08 至 2022-11，共 112 个月。

### 字段说明

| 字段 | 含义 | 缺失数 | 备注 |
|---|---|---|---|
| `code` | 股票代码 | 0 | 深市A股，如 `000568.SZ`，共 261 只 |
| `close` | 月末收盘价 | 8,422 | 大概率不复权（需 Phase 0 验证），缺失主要为未上市 |
| `divi` | 股息率快照（%） | 14,964 | 百分比形式，几乎每月更新；`divi = 固定DPS / 当前close * 100` |
| `cap` | 总市值 | 7,444 | 997 行为 0（对应未上市），19 行缺失但 close 有效 |
| `date` | 日期 | 0 | 格式 `YYYYMMDD`，均为月初 |

### divi 语义确认（经实证验证）

| 观察 | 实证结果 | 结论 |
|---|---|---|
| `dps = close * divi / 100` 相邻变化 | 中位变化 = 0.00，98% 变化 < 0.01 | DPS 在分红更新之间恒定 |
| `close` 变化与 `divi` 变化相关性 | 多数个股内显著负相关 | `divi = 固定DPS / 变动close`，是活股息率快照 |
| divi 非空频率 | 109/112 月（泸州老窖） | divi 不是事件标识，是连续更新的快照 |

**核心含义**：`divi` 是每月更新的股息率快照（`固定年分红 / 当前股价`），不是"当月发生了分红"。因此：
- 因子构建：`dps_proxy = close * divi / 100` → 按股票 ffill → `dy = dps_proxy_ffill / close * 100`（不能直接 ffill divi，会冻结旧股价下的股息率）
- 有效样本中 divi 原始覆盖率约 68.6%，经 dps ffill 后约 84.9%
- 收益计算：**不能**从 divi 重建含息收益——无除息日期信息，强行加会假设每月除息一次
- 报告中应坦诚此限制

### 样本池的动态变化

**CSV 每月固定 261 行，但可交易样本远非固定**：
- close 非空且 cap>0 的有效样本：从 2013-08 的 **135 只** 逐步增长到 2022-11 的 **259 只**
- 缺失主因为尚未上市（每年逐步新增 6-25 只），非停牌
- 回测时必须每月按 `close.notna() & (cap > 0)` 动态筛选

### divi 分布（非空样本）
- 中位数 0.86%、75 分位 1.49%、99 分位 6.80%、最大 8.46%
- 236 只股票有至少一次分红记录，25 只从未分红

## 三、Phase 0：数据口径核验（必须最先做）

1. **close 是否复权（关键）**：对比同股票在已知除息日前后价格。大概率是不复权价格
2. **divi 口径最终确认**：已有实证支持"divi = 固定DPS/当前close 的快照"，在 notebook 中可视化验证
3. **收益定义决策**：
   - 由于无法确认实际除息日，**主收益口径使用价格收益**：`price_ret = close_{t+1} / close_t - 1`
   - 在报告中明确声明：收益不含分红，高分红股票在除息月的实际收益被略微低估
   - 这实际上**更保守**：如果真的找到红利溢价，说明信号不是靠"数钱数出来的"
4. **cap=0 样本处理**：cap=0 或 cap 缺失的直接剔除
5. **每月有效样本数量图**（close 非空且 cap>0）
6. **幸存者偏差检查**：确认数据是否含退市股；如排除，报告中声明

## 四、本次策略方向：消费红利因子选股

### 4.1 主题定位

"消费红利" = 消费行业 + 红利因子。以股息率为主因子，辅以动量/反转/低波/规模因子。

### 4.2 可实现的候选因子

**红利类**（构建方式：`dps_proxy = close * divi / 100` → ffill → 重新除以最新 close）：

| 因子 | 构建方式 | 经济逻辑 |
|---|---|---|
| 股息率（DY） | `dps_proxy_ffill / close * 100` | 最近已知 DPS / 当前股价，活的股息率 |
| 红利稳定性 | 过去 5 年中 divi > 0 的月份占比 | 持续分红能力 > 一次性高分红 |
| DPS 增长 | `dps_proxy_ffill / dps_proxy_ffill.shift(12) - 1` | 每股分红同比增速，纯红利质量信号 |
| DY 变化 | `dy / dy.shift(12) - 1` | 股息率同比变化 = DPS 变化 + 股价变化综合效应 |

> **为什么不能直接 ffill divi**：直接 ffill 会冻结旧股价对应的股息率（价格已变但 yield 不变）。应还原 DPS 代理 → ffill DPS → 重新除以最新 close。实证中 10.5% 的行两种方法差异 > 0.1%，最大差 7.55%，对横截面排序有实际影响。

**估值与规模类**：

| 因子 | 构建方式 | 经济逻辑 |
|---|---|---|
| 规模 | `ln(cap)` | 小盘股溢价效应 |

**动量与低波类**（从月频价格收益序列构建）：

| 因子 | 构建方式 | 经济逻辑 |
|---|---|---|
| 短期反转 | 最近 1 月价格收益 | A 股短期反转效应 |
| 中期动量 | 过去 6 月累计价格收益 - 1 | 中期趋势延续 |
| 长期反转 | 过去 12 月累计价格收益 - 1 | 长期反转效应 |
| 低波动 | 过去 12 月月度收益标准差 | 低波异象 |

> **关于价格收益的偏差**：高分红股票在除息月价格跌幅更大，因此动量/低波因子如基于价格收益会略微低估高分红股。这一偏差方向已知（低估高分红股的正收益），应在报告中作为口径限制说明。好在 A 股一年通常只集中分红一次，影响集中在少数月份，因子排序的整体稳健性尚可。

### 4.3 不可实现的因子（明确排除）

| 伪因子 | 排除原因 |
|---|---|
| EP、BM | 无净利润/净资产数据 |
| 日度低波/换手率 | 无日频数据、无成交额字段 |
| 含息总收益因子 | 无除息日数据，无法精确重建 |

### 4.4 策略设计思路

1. **核心**：股息率（DY）——高分红消费股是策略的锚
2. **补充**：红利稳定性 + DPS 增长 + DY 变化，刻画红利质量而非数量
3. **辅助**：动量/反转/低波与红利相关性较低，提供截面增量信息
4. **规模控制**：`ln(cap)` 作为控制变量或纳入合成

## 五、回测协议

### 5.1 时间对齐与收益定义
- t 月月末已知信息 → 预测 t+1 月收益
- **主收益口径**：`price_ret = close_{t+1} / close_t - 1`（价格收益，不含分红）
- 报告中明确声明此口径及其影响方向
- 严禁任何未来函数

### 5.2 投资域
- 每月 `close.notna() & (cap > 0)` 动态确定可交易池

### 5.3 基准
- 可交易样本等权基准（价格收益口径，不计手续费）
- 可选市值加权对照

### 5.4 调仓频率
- 主策略：月度调仓
- 稳健性：季度调仓对照

### 5.5 组合构建
- 按综合评分排序选 Top N
- 等权（主口径）与排名加权（对照）
- 月频调仓下缓冲意义有限，可仅做无缓冲 TopN

### 5.6 费用
- 交易费率：`fee_rate = 0.001`（单边千分之一）
- 每期交易成本：`cost = 2 × turnover × fee_rate`（买入 + 卖出各付一次）
- 稳健性：`0`、`0.002`、`0.003`
- 换手率：`turnover = (买入额 + 卖出额) / (2 × 期初组合总资产)`
- 由于收益口径为价格收益（不含分红），**红利税不纳入回测**。报告中注明：若考虑红利再投资，实际收益率会略高，但月度调仓会触发 20% 红利税，部分抵消

### 5.7 绩效指标
年化收益、年化波动、夏普比率、最大回撤、Calmar 比率、超额累计收益、信息比率、平均换手率、月度胜率

### 5.8 A 股红利策略常见陷阱
1. **收益口径偏差**：价格收益低估高分红股真实回报（保守偏差，可接受）
2. **红利税**：若实际执行含息策略，月度调仓触发 20% 红利税
3. **行业集中度**：消费行业内高分红集中在白酒、乳业等子行业，策略可能被动行业押注
4. **红利陷阱**：高股息率可能来自股价暴跌，需结合质量/动量过滤
5. **样本容量**：早期仅 ~135 只有效股票，Top10 集中度很高
6. **幸存者偏差**：若数据排除退市股，回测收益偏高
7. **小盘效应混淆**：A 股高股息与小市值有相关性，建议做规模中性化检验

## 六、实施步骤

### Phase 0：数据口径核验（见第三节）

### Phase 1：数据预处理
1. 过滤 `cap > 0` 且 `close.notna()` 的样本
2. 构建红利因子：
   - `dps_proxy = close * divi / 100`（仅 divi 非空月）
   - `dps_proxy_ffill = dps_proxy.groupby(code).ffill()`（首次出现分红记录前保持 NaN 或填 0，**严禁 bfill**——会用未来分红信息）
   - `dy = dps_proxy_ffill / close * 100`
3. 构建 `price_ret = close.shift(-1) / close - 1`（注意最后一行 NaN）
4. 构建其余因子面板（规模、动量/反转/低波等）

### Phase 2：单因子检验
1. 分组回测：每月按因子分 5 组，等权持有，G1-G5 + long_short 净值图
2. IC/RankIC/IR 统计表
3. 因子去留总表（经济逻辑 + 视觉证据 + 统计证据）

### Phase 3：多因子合成
1. 因子相关性热力图
2. 去极值（MAD 或分位数）→ z-score → 方向统一 → 等权平均
3. 不同因子组合版本的绩效对比

### Phase 4：策略回测
按第五节协议执行，输出主策略 vs 基准净值图、超额净值图、绩效指标表、多空对冲验证

### Phase 5：稳健性检验
1. 调仓频率（月频 vs 季频）
2. TopN 敏感性（10/20/30）
3. 手续费敏感性（0 到 0.003）
4. 年度表现 + 分阶段表现
5. 滚动验证（3 年训练 + 1 年验证）
6. 参数网格扫描（仅观察，不选最优）

### Phase 6：报告撰写
1. Markdown 报告（摘要 + 正文 + 附录）
2. 图片导出到 assets/
3. 可选：Word 报告生成
